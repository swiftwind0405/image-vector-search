# QNAP 上的图片向量检索服务（Milvus + Jina API）

**目标**：和 agent 对话时，agent 能从向量数据库里检索出匹配的本地图片（几万张）。  
**约束**：
- 走 **API**（向量化用 Jina），不本地跑大模型
- 运行在 **QNAP**（Container Station）
- **全手动**：不做定时任务；需要时手动执行索引 / 搜索
- 代码放 GitHub；GitHub Actions 构建 Docker 镜像到 **GHCR**（公开）；QNAP 用 `docker-compose.yml` 一键部署
- 所有敏感参数通过环境变量注入（`JINA_API_KEY` 等）

---

## 1) 总体架构

```
┌───────────────┐
│ 你 / 对话 Agent │
└───────┬───────┘
        │ 查询文本
        │
        ▼
┌──────────────────────────────┐
│ QNAP 容器：milvus-image-search │
│  - Milvus Standalone (向量库)  │
│  - scripts/*.py (索引/检索)    │
└───────┬──────────────────────┘
        │
        | 调 Jina API 获取 embedding（文本/图片）
        |
        ▼
  返回 TopK 图片路径（位于 /data/images 映射的 NAS 目录）
```

**本方案公开镜像，无需 GHCR 登录**。所有敏感参数通过环境变量注入。

**数据落盘**：
- 图片文件：QNAP 共享目录（如 `/share/Photos`）
- 向量库数据：QNAP 目录（如 `/share/Container/milvus-image-search/milvus-data`）

## 3) 镜像构建与发布（GitHub Actions → GHCR）

### 3.1 公开镜像设置

**无需 GHCR 登录**，设置镜像为公开即可：

1. GitHub Repo → **Settings → Actions → General**：允许 Actions 运行
2. Repo → **Settings → Packages** → 点击你的 package → **Make public**
3. 或者在 `workflow` 里加 `visibility: public`（见下方）

### 3.2 `.github/workflows/build.yml`

```yaml
name: Build and Push Docker Image

on:
  push:
    branches: [ main ]
  workflow_dispatch:

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=raw,value=latest
            type=sha,format=short

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          visibility: public
```

发布后镜像地址形如：
- `ghcr.io/stanley-wind/milvus-image-search:latest`

> QNAP 直接 `docker pull ghcr.io/stanley-wind/milvus-image-search:latest` 即可，无需登录。

## 4) QNAP 部署（Container Station + docker-compose.yml）

### 4.1 QNAP 目录规划（建议）

- 图片目录：`/share/Photos`（按你的实际）
- Milvus 数据：`/share/Container/milvus-image-search/milvus-data`

在 QNAP 上创建目录：

```bash
mkdir -p /share/Container/milvus-image-search/milvus-data
```

### 4.2 `docker-compose.yml`

> QNAP Container Station 支持用 compose 创建应用（不同型号 UI 有差异；本质是同一个 compose）。

```yaml
services:
  milvus-image-search:
    image: ghcr.io/stanley-wind/milvus-image-search:latest
    container_name: milvus-image-search
    restart: unless-stopped

    # 端口：使用冷门端口避免冲突（只在内网使用即可）
    ports:
      - "25330:19530"  # Milvus gRPC（宿主机映射到 25330）
      - "29091:9091"   # Milvus HTTP/health（宿主机映射到 29091）

    environment:
      # 你的 Jina API key（建议在 QNAP UI 里用 Secret/环境变量配置）
      - JINA_API_KEY=${JINA_API_KEY}
      # 图片目录（容器内路径）
      - IMAGE_DIR=/data/images
      # Milvus 连接（容器内直连本机 standalone）
      - MILVUS_HOST=127.0.0.1
      - MILVUS_PORT=19530
      - MILVUS_HTTP_PORT=9091
      - COLLECTION_NAME=${COLLECTION_NAME:-image_search}

    volumes:
      # 图片库只读挂载（推荐）
      - /share/Photos:/data/images:ro
      # Milvus 数据落盘
      - /share/Container/milvus-image-search/milvus-data:/var/lib/milvus
```

> 如果你希望容器内脚本能写入图片侧的元数据（不推荐），可以去掉 `:ro`。

**环境变量说明**（在 QNAP Container Station 应用创建页填写）：
- `JINA_API_KEY`：你的 Jina API Key
- `COLLECTION_NAME`：（可选）默认 `image_search`

### 5.2 `entrypoint.sh`

```bash
#!/bin/bash
set -e

MILVUS_HTTP_PORT=${MILVUS_HTTP_PORT:-9091}

echo "[milvus-image-search] starting Milvus standalone..."
/opt/milvus/bin/milvus run standalone &
MILVUS_PID=$!

echo "[milvus-image-search] waiting for Milvus healthz..."
for i in {1..60}; do
  if curl -fsS http://127.0.0.1:${MILVUS_HTTP_PORT}/healthz >/dev/null 2>&1; then
    echo "[milvus-image-search] Milvus is ready."
    break
  fi
  sleep 1
  if [ "$i" -eq 60 ]; then
    echo "[milvus-image-search] Milvus did not become ready in time." >&2
    exit 1
  fi
done

# 容器常驻（你手动 docker exec 运行脚本）
# 使用 exec 确保能正确处理 SIGTERM
exec tail -f /dev/null
```

## 6) Python 脚本（索引 / 搜索）

### 6.1 公共约定

- Embedding 模型：`jina-clip-v1`
- 向量维度：**768**（Jina CLIP 默认）
- 相似度：**COSINE**
- collection：`image_search`
- 字段：
  - `path`：图片路径（字符串）
  - `embedding`：向量

脚本通过环境变量读取：
- `JINA_API_KEY`（必填）
- `IMAGE_DIR`（默认 `/data/images`）
- `MILVUS_HOST`（默认 `127.0.0.1`）
- `MILVUS_PORT`（默认 `19530`）
- `MILVUS_HTTP_PORT`（默认 `9091`，仅 health check 用）
- `COLLECTION_NAME`（默认 `image_search`）

### 6.2 `scripts/index_images.py`（批量索引）

```python
#!/usr/bin/env python3
import os
import sys
import time
import base64
import argparse
from pathlib import Path
from typing import List

import requests
from tqdm import tqdm
from pymilvus import (
    connections,
    FieldSchema, CollectionSchema, DataType,
    Collection, utility,
)

JINA_API_KEY = os.environ.get("JINA_API_KEY", "")
IMAGE_DIR = os.environ.get("IMAGE_DIR", "/data/images")
MILVUS_HOST = os.environ.get("MILVUS_HOST", "127.0.0.1")
MILVUS_PORT = os.environ.get("MILVUS_PORT", "19530")
COLLECTION_NAME = os.environ.get("COLLECTION_NAME", "image_search")

MODEL = "jina-clip-v1"
DIM = 768
BATCH = 64

EXTS = {".jpg", ".jpeg", ".png", ".webp"}


def jina_embed_image(paths: List[str]) -> List[List[float]]:
    if not JINA_API_KEY:
        raise RuntimeError("JINA_API_KEY is empty")

    inputs = []
    for p in paths:
        with open(p, "rb") as f:
            b64 = base64.b64encode(f.read()).decode("utf-8")
        # 统一按 jpeg data url（实际内容无所谓）
        inputs.append({"image": f"data:image/jpeg;base64,{b64}"})

    r = requests.post(
        "https://api.jina.ai/v1/embeddings",
        headers={
            "Authorization": f"Bearer {JINA_API_KEY}",
            "Content-Type": "application/json",
        },
        json={"model": MODEL, "input": inputs},
        timeout=120,
    )
    r.raise_for_status()
    data = r.json()["data"]
    # 保持顺序
    data.sort(key=lambda x: x["index"])
    return [row["embedding"] for row in data]


def get_or_create_collection() -> Collection:
    connections.connect(host=MILVUS_HOST, port=MILVUS_PORT)

    if utility.has_collection(COLLECTION_NAME):
        col = Collection(COLLECTION_NAME)
        return col

    fields = [
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(name="path", dtype=DataType.VARCHAR, max_length=1024),
        FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=DIM),
    ]
    schema = CollectionSchema(fields, description="Local image search")
    col = Collection(name=COLLECTION_NAME, schema=schema)

    index_params = {
        "metric_type": "COSINE",
        "index_type": "IVF_FLAT",
        "params": {"nlist": 1024},
    }
    col.create_index(field_name="embedding", index_params=index_params)
    return col


def main():
    parser = argparse.ArgumentParser(description="Index images to Milvus")
    parser.add_argument(
        "--folder",
        type=str,
        default=None,
        help="Only index images under this subfolder (relative to IMAGE_DIR)",
    )
    args = parser.parse_args()

    col = get_or_create_collection()

    root = Path(IMAGE_DIR)
    if not root.exists():
        raise RuntimeError(f"IMAGE_DIR not found: {IMAGE_DIR}")

    if args.folder:
        root = root / args.folder
        if not root.exists():
            raise RuntimeError(f"Subfolder not found: {root}")

    image_paths = [str(p) for p in root.rglob("*") if p.suffix.lower() in EXTS]
    image_paths.sort()

    print(f"Found {len(image_paths)} images under {root}")

    to_insert_paths = []
    to_insert_vecs = []

    for i in tqdm(range(0, len(image_paths), BATCH), desc="Indexing"):
        batch_paths = image_paths[i : i + BATCH]
        try:
            vecs = jina_embed_image(batch_paths)
        except requests.exceptions.HTTPError as e:
            if e.response.status_code == 429:
                print(f"Rate limited, waiting 5s...")
                time.sleep(5)
                vecs = jina_embed_image(batch_paths)
            else:
                raise
        to_insert_paths.extend(batch_paths)
        to_insert_vecs.extend(vecs)

        if len(to_insert_paths) >= 500:
            col.insert([to_insert_paths, to_insert_vecs])
            to_insert_paths, to_insert_vecs = [], []
            # 避免 Jina API 限流，批次间短暂停顿
            time.sleep(0.5)

    if to_insert_paths:
        col.insert([to_insert_paths, to_insert_vecs])

    col.flush()
    col.load()
    print("Done. Entities:", col.num_entities)


if __name__ == "__main__":
    main()
```

**用法**：
```bash
# 索引全部图片
python /opt/scripts/index_images.py

# 只索引某个子文件夹（如 /data/images/2024/）
python /opt/scripts/index_images.py --folder 2024
```

### 6.3 `scripts/search_images.py`（文本搜图）

```python
#!/usr/bin/env python3
import os
import sys
from typing import List

import requests
from pymilvus import connections, Collection

JINA_API_KEY = os.environ.get("JINA_API_KEY", "")
MILVUS_HOST = os.environ.get("MILVUS_HOST", "127.0.0.1")
MILVUS_PORT = os.environ.get("MILVUS_PORT", "19530")
COLLECTION_NAME = os.environ.get("COLLECTION_NAME", "image_search")

MODEL = "jina-clip-v1"


def jina_embed_text(text: str) -> List[float]:
    if not JINA_API_KEY:
        raise RuntimeError("JINA_API_KEY is empty")

    r = requests.post(
        "https://api.jina.ai/v1/embeddings",
        headers={
            "Authorization": f"Bearer {JINA_API_KEY}",
            "Content-Type": "application/json",
        },
        json={"model": MODEL, "input": [text]},
        timeout=60,
    )
    r.raise_for_status()
    return r.json()["data"][0]["embedding"]


def main():
    if len(sys.argv) < 2:
        print("Usage: search_images.py <query_text> [top_k]")
        sys.exit(1)

    query = sys.argv[1]
    top_k = int(sys.argv[2]) if len(sys.argv) >= 3 else 5

    connections.connect(host=MILVUS_HOST, port=MILVUS_PORT)
    col = Collection(COLLECTION_NAME)
    col.load()

    qv = jina_embed_text(query)

    res = col.search(
        data=[qv],
        anns_field="embedding",
        param={"metric_type": "COSINE", "params": {"nprobe": 32}},
        limit=top_k,
        output_fields=["path"],
    )

    for rank, hit in enumerate(res[0], start=1):
        path = hit.entity.get("path")
        score = hit.distance
        print(f"{rank}	{score:.6f}	{path}")


if __name__ == "__main__":
    main()
```

## 9) 关键注意事项

1. **JINA_API_KEY 不要提交到 GitHub**：只放在 QNAP 的环境变量/Secret。
2. **图片挂载只读**：`/share/Photos:/data/images:ro`，避免脚本误写。
3. **向量库持久化**：Milvus 数据务必映射到 NAS 目录，否则容器重建会丢。
4. **几万张图片首轮索引耗时**：主要取决于 Jina API 调用速度与网络；可以分批做（按文件夹）。
5. **Jina API 限流**：免费/低 tier 账号可能有 rate limit，`index_images.py` 已内置 429 重试和批次间 sleep。
6. **隐私**：图片会被上传到 Jina API 做 embedding（如果你介意隐私，需要改成本地模型）。
